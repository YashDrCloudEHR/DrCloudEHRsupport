DrCloudEHR : Casamba Documents

Created by Tejasvini Aurangabadkar, last modified on Mar 28, 2018

  • Connecting to Casamba Server

The resident care plan documents are uploaded on to Casamba by Ergo on a regular basis. Each care plan consists of a set of two files.

  • XML file
  • PDF file

When a request is initiated from DrCloudEMR, it downloads all the care plan documents from the Casamba site and assigns them to the corresponding resident(s) in DrCloudEMR using certain preset matching criteria.

Connecting to Casamba Server

  1. Navigate to Practice > Administrative > Practice.
  2. Open the Practice Settings page.
  3. Click the Casamba link.
  4. Casamba Connection Information page opens.
  5. Click on the default facility link.

Casamba Connection Information page can be edited from here.

  1. Facility: Select the facility from the list box of facilities provided. The selected facility is the facility for which the following connectivity information applies.
  2. Host URL:Host Url is the SMTP location on Casamba where Ergo uploads documents to.
  3. User Id: User id and password provided by Casamba goes in here.
  4. Remote Folder: Remote Folder is the folder on Casamba where Ergo uploads the documents to.
  5. Download Folder: Download Folder is the folder path on DrCloudEMR to download all the data related to your facility from Casamba . This is essentially a staging area for DrCloudEMR.
  6. Matching Criteria: Matching Criteria is the criteria that will be used to assign the downloaded care plan to the resident in DrCloudEMR.
  7. Criteria options are provided as option buttons:
    • SSN(Match by Social Security Number)
    • Fname and Lname and DOB(Match by First Name, Last Name and Date Of Birth)
  8. Delete on Download: Enabling this checkbox auto deletes the downloaded files from the Casamba server after download is completed to DrCloudEMR. This is to prevent successfully downloaded documents from being downloaded again.

Casamba Documents Download

  1. Navigate to Practice > Administrative > Egro-Casamba Docs.
  2. Ergo Casamba Documents page opens.
  3. Select the facility from the dropdown ,which is mapped from the Casamba Connection Information page.
  4. Click on Download button to download all the documents to DrCloudEMR from Casamba and assign them to the residents in DrCloudEMR according to the matching criteria set on the Casamba Connection Information page.
  5. Once a document is downloaded it is in one of the two states:
    • Matched:These are the documents that were matched and assigned to residents in DrCloudEMR.The matched results are shown in black color.
    • Unmatched:These are the documents that were unmatched and hence not assigned to any resident(s) in DrCloudEMR. The unmatched results are displayed in red color.
    • The Unmatched documents can be manually matched to residents in DrCloudEMR as follows:
      • Select all the checkboxes for the unmatched residents.
      • Click on Assign.
      • Map each unmatched resident to a resident in DrCloudEMR in the "Casamba Reports - Assign Residents" dialog box.
      • Close the dialog box.
  6. The assigned resident records are now shown in black, indicating matched status.

Casamba Results Delete

This enables us to search the downloaded documents on DrCloudEMR based on certain search filters.

  1. Navigate to Practice > Administrative > Egro-Casamba Docs.
  2. Ergo Casamba Documents page opens.
  3. Find the search section.
  4. Select the Resident Name search option from the dropdown.
  5. Enter the search criteria of the selected option.
    • Matched: Displays all the downloaded documents from Casamba that were matched and assigned to residents in DrCloudEMR.
    • Unmatched: Displays all the downloaded documents from Casamba that were unmatched and hence not assigned to any resident(s) in DrCloudEMR.
  6. Select the checkboxes and click on the Delete button. This will delete the selected document(s) from DrCloudEMR.
