DrCloudEHR : Clinical Process Configuration

Created by Dan Hynes on Aug 20, 2020

Add the facilities, document workflows, service code/modifier combinations, and encounter categories you use when performing various clinical and billing procedures to DrCloudEHR.

This topic contains the following sections:

  • Define Facilities
  • Define Service Codes
  • Define Workflows
  • Create Categories

Define Facilities

Add information about each facility or program you use to provide treatment. You select the facilities or programs where services are billed or provided when you perform various tasks in DrCloudEHR.

Once you add a facility, you cannot delete it. You can make a facility inactive in the system, which prevents users from selecting it.

  1. Click the Practice tab. 
  2. On the left navigation menu, click the Administrative tab, and then select Facilities.
  3. Click Add. The Add Facility dialog box opens.
  4. Click the Address and Remit To tabs and enter the necessary details.
  5. Click the Additional Info tab and provide relevant business details.
    • If the facility is a Medicare Administrative Contractor (MAC), enter the Provider Transaction Access Number (PTAN) in the Medicare PTAN field.. 
    • If the facility uses taxonomy codes used to identify the provider specialty at the claim level, enter the 10 character alphanumeric code.
  6. Click the Settings tab and provide the billing details. 
    • If the facility is the billing location, select the Accepts Assignments check box. If the box is not selected, electronic signatures are not set in Box 13 in HCFA forms submitted electronically.
    • Enter contact details for the primary billing contact for the facility.
  7. Click Save to save the facility.

Define Service Codes

Define the service code and modifier combinations used by your clinic in DrCloudEHR. You select the codes when you perform various clinical and billing tasks.

  1. Click the Practice  tab. 
  2. On the left navigation menu, click the Administrative tab, and then select Services.
  3. Select the code type, and then enter the service code. 
  4. If applicable, enter up to four modifier codes. To create a new code based on an existing code, must add a new modifier to prevent duplicates.
  5. Select the appropriate checkbox to indicate if the service should be included in diagnosis or service reports. Service codes are not included in reports by default.
  6. If you select REV as the code type, optionally click Add to specify the related service.
  7. If you select DSM-5, FunctStatus, or POV as the code type, the Mapped ICD Code field appears. Select the checkbox to indicate whether the mapped code should be included on the UBO4 claim form instead of the code entered, and then enter the mapped code to include in the form in the Mapped ICD Code field. 
  8. Enter your standard fee amount for the service. Set the amount to a value 10% to 20% higher than the amount specified in your contract with the insurance company. Insurance companies typically pay the lowest possible amount, so set the fee amount higher to ensure you receive the best payment. 
  9. Click Add as New to save the service definition.

Define Workflows

A workflow is a collection of encounter and consent forms used in a specific encounter type. You select the forms to include, and specify the order in which they appear in an encounter.

When you create a category for the encounter type, you specify the workflow containing the forms used with encounters. When you create a new encounter and select a category that includes the workflow, the forms in the workflow display in the encounter window, in the order specified in the workflow.

Note that you cannot change the name of a workflow. If you want to assign a new name, delete the workflow, and then re-create it.

  1. Click the Practice tab. 
  2. On the left navigation menu, click the Administrative tab, and then select Forms Workflow.
  3. Click Add New Workflow button. The Add New Workflow dialog box opens.
  4. Enter a name and description for the workflow.
  5. Select each program or facility where the appointments occur.
  6. Select the Show at Forms Cabinet checkbox to include the forms in the Forms Cabinet. 
  7. Click Save. The new workflow appears in the table on the Forms Workflow page.
  8. Locate the workflow in the table, and then click the Forms link in the Add Forms column. 
  9. Select the forms to include, then use the arrow buttons to move the forms into the Forms Selected window. 
  10. Use the Up and Down keys to specify the order in which forms appear in the encounter summary page.
  11. Enter the order in which the workflow appears in the Forms Cabinet in the Priority field. Set the value to 0 to have the workflow display first in the list.

Create Categories

A category is a template used for a specific encounter or appointment type. The category contains codes, facilities, providers, and workflows specific to the encounter. When a new encounter is created, the provider selects the category to use. Dependent on data already defined in the system, such as service codes, facilities, and workflows. 

When a user schedules a new appointment in the calendar, they select the category. 

  1. Click the Practice  tab. 
  2. On the left navigation menu, click the Administrative tab, and then select Categories.
  3. Click Add.
  4. Select the checkbox indicating payment type.
  5. Enter a name and description that map to the encounter type.
  6. Indicate whether the category applies to patients, providers, or a group of patients that receive the same service.
  7. Click the Click to choose the code  link to select service codes and modifiers defined in the system to associate with the category. You can select multiple codes.
    If you add codes manually, format codes as type:number-modifier. For example: CPT4:90834-HF. Separate each code with a comma.
  8. Select the Place of Service (POS) where the appointment typically occurs.
  9. Specify the typical duration of appointments of this type. This data is useful when scheduling appointments in the calendar.
  10. Optionally specify the color to use for this type of appointment.
  11. Select the workflow that contains encounter forms typically used for this encounter type.
  12. Select the facility and a list of providers associated with the encounter type.
  13. Optionally indicate how often the encounter repeats. It is advisable not set this property, but instead schedule appointments as needed in the calendar.
  14. Click Save to save the category.
