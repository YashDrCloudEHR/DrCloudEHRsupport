DrCloudEHR : Using Quality Management Rules

Created by Dan Hynes on Aug 12, 2020

DrCloudEHR provides quality management features that enable you to ensure that forms used in encounters comply with internal and external procedures and regulations when billing claims. Quality management features help you avoid potential issues when undergoing an audit or when submitting claims to insurance providers. 

You can define rules, known as Golden Thread rules, to require that specific forms be completed for selected facilities or programs, as well as the frequency or time frame for which the forms must be completed. You can also define form completion criteria requiring the completion of specific fields in a form based on business, clinical, regulatory and workflow requirements. Golden Thread rules and form completion rules work in tandem to ensure the documentation associated with an encounter is complete. 

You can configure Golden Thread and encounter form completion rules to prevent users from submitting claims if the rules are not satisfied.

This topic contains the following sections:

  • Define Encounter Form Completion Criteria
  • Add a Golden Thread Rule

Define Encounter Form Completion Criteria

You can define rules specifying that specific fields be filled out in an encounter form for the form to be considered complete. 

  1. Click the Practice tab.
  2. Click the Administration tab, and then click Forms Settings.
  3. Click a form name in the table. 
  4. Click Add.
  5. Enter the starting date on which the rule takes effect.
    You can optionally enter an end date. If you do not specify an end date, the rule remains in effect.
  6. Select Block from being billed to prevent users from submitting a claim if the rule is not satisfied.
  7. Select the checkbox for each field in the form you want to require be completed.
  8. Click Save.

Add a Golden Thread Rule

Define rules requiring specific forms be completed for selected facilities or programs, a specific time frame, or for an encounter category.

When you define a rule, you can require that selected encounter forms meet the form completion rules defined in the Forms Settings page.

Note: The Value and Advanced Golden Thread rule types are reserved for the DrCloudEHR product development team.

To add a new golden thread rule, do the following:

  1. Click the Practice tab.
  2. In the left hand navigation menu, click the Golden Thread tab, and then click Golden Thread Rules. The Golden Thread Rules page opens.
  3. Click the Add Golden Thread Rule button.
  4. Enter a name for the rule.
  5. Select Form as the rule type.
  6. Enter the starting date on which the rule takes effect. You can optionally enter an end date. If you do not specify an end date, the rule remains in effect.
  7. Select the facilities or programs to which to apply the rule. 
    Specify the AND operator to apply the rules to all selected facilities.​​ Use the OR operator to apply the rule to at least one facility.
  8. Select the forms that must be submitted. 
    Specify the AND operator to require a user to complete all of the selected forms.​​ Use the OR operator to require that at least one of the selected forms be completed.
  9. Select the frequency at which the forms must be completed.
  10. Select Must meet the form completion criteria to require that the selected encounter forms meet any form completion criteria.
    Form completion criteria are defined in the Forms Settings page. For more information, see Define Encounter Form Completion Criteria.
  11. Specify the message to display in notifications if the user fails to follow the rule. 
  12. Select Block from being billed to prevent users from submitting a claim if the rule is not complete. 
  13. Click the Notifications tab in the Add Golden Thread Rule dialog, and then configure how and to whom notifications are sent if the rule is violated. 
    If you select Billing Manager, a message appears with an alert icon in the Billing Manager to indicate that the rule is not satisfied.
  14. Click the Exceptions tab in the dialog, and then select the categories to ignore when applying the rule.
  15. Click Save.
